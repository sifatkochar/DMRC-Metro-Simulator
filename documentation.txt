CONSIDERATIONS/ASSUMPTIONS regarding the CODE -

The function data_func() helps gather all the data from the 'metro_data.txt' file.
It stores all the data of the file in the form of a dictionary, with its keys being the 5 headings of the text file.
The value of a key is a list containing each line under that heading as an individual dictionary.
The keys of these sub-dictionaries are the sub-headings that were mentioned in the # rows. The values correspond to the respective values for each row under these sub-headings.

For the stations order -
Blue (L3) -- Forward Direction = DWARKA SECTOR - 21 to NOIDA ELECTRONIC CITY
Blue (L4) -- Forward Direction = DWARKA SECTOR - 21 to VAISHALI
Magenta -- Forward Direction = JANAKPURI WEST to BOTANICAL GARDEN
Yellow -- Forward Direction = SAMAYPUR BADLI to MILLENNIUM CITY CENTRE GURUGRAM

All station names/line names to be inputted in CAPITALS.
Times to be inputted in the format - "10:28"

The functions - from_to_time(), route_shortest_time() and best_route_time() together work to operate option 2 of the menu.
I tried to use the concept of 'Breadth-First Search' to compute this.
I referred to GeeksForGeeks to learn a little about this topic.

https://www.geeksforgeeks.org/dsa/breadth-first-search-or-bfs-for-a-graph/

Option 2 of the menu functions well by providing the path/routing for current stations and destinations on the same line or different lines. It also lets the user know if their start and end stations are on the same line or not.

CONSIDERATION -
Option 2 of the menu, when selected - displays the complete path and time taken between any two stations on same or different lines. It also mentions if the stations lie on the same line or not.
It however works while considering that the user does not require the number of interchanges taking place.
